## Module shape — `api/upload-leads.js`

Vercel Node.js serverless function (modern `@vercel/node` style).

```javascript
// api/upload-leads.js
import { put } from '@vercel/blob';

const BLOB_KEY = 'leads-current.json';
const MAX_BODY_BYTES = 5 * 1024 * 1024; // 5 MB; M7's output is ~100 KB
const REQUIRED_NICHES = ['it_msp', 'mssp', 'cloud'];

export default async function handler(req, res) {
  // Method gate
  if (req.method !== 'POST') {
    res.setHeader('Allow', 'POST');
    return res.status(405).json({ error: 'method not allowed' });
  }

  // Auth
  const expected = process.env.LEADS_UPLOAD_API_KEY;
  if (!expected) {
    return res.status(500).json({ error: 'server misconfigured: no API key' });
  }
  const auth = req.headers.authorization || '';
  if (auth !== `Bearer ${expected}`) {
    return res.status(401).json({ error: 'unauthorized' });
  }

  // Body shape — light validation only
  const body = req.body;
  if (!body || typeof body !== 'object') {
    return res.status(400).json({ error: 'body must be JSON object' });
  }
  if (!body.niches || typeof body.niches !== 'object') {
    return res.status(400).json({ error: 'missing niches object' });
  }
  for (const niche of REQUIRED_NICHES) {
    if (!Array.isArray(body.niches[niche])) {
      return res.status(400).json({ error: `niches.${niche} must be array` });
    }
  }

  // Write to Blob
  try {
    const json = JSON.stringify(body);
    if (Buffer.byteLength(json, 'utf8') > MAX_BODY_BYTES) {
      return res.status(413).json({ error: 'payload too large' });
    }
    const blob = await put(BLOB_KEY, json, {
      access: 'public',
      contentType: 'application/json',
      addRandomSuffix: false, // overwrite stable key
      allowOverwrite: true,
    });
    return res.status(200).json({
      ok: true,
      url: blob.url,
      uploaded_at: new Date().toISOString(),
      total_leads: Object.values(body.niches).reduce(
        (acc, arr) => acc + arr.length, 0,
      ),
    });
  } catch (err) {
    console.error('blob put failed:', err);
    return res.status(500).json({ error: 'blob upload failed' });
  }
}

// Increase Vercel's default body parser limit (default 1 MB).
export const config = {
  api: { bodyParser: { sizeLimit: '5mb' } },
};
```

Vercel's auto-routing makes this reachable at `<deployment>/api/upload-leads`.

## `daily_run.py` — `--upload` addition

A new flag that POSTs the just-written JSON to `LEADS_UPLOAD_URL` with
the bearer token. Runs after `_build_output(...)` and the JSON write.

```python
# In msp_pipeline/daily_run.py

import os
import requests  # already a transitive dep via M2 sources

# Existing imports + module-level constants...

def main(argv: list[str] | None = None) -> int:
    # ... existing pipeline ...
    args.output_path.write_text(json.dumps(output, indent=2, default=str))
    log.info("wrote %s", args.output_path)

    if args.upload:
        try:
            _upload_blob(args.output_path)
        except Exception:
            log.exception("upload failed")
            return 1  # non-zero exit so M10 cron sees the failure
    return 0


def _upload_blob(json_path: Path) -> None:
    url = os.environ.get("LEADS_UPLOAD_URL")
    api_key = os.environ.get("LEADS_UPLOAD_API_KEY")
    if not url or not api_key:
        raise RuntimeError(
            "--upload requires LEADS_UPLOAD_URL and LEADS_UPLOAD_API_KEY env vars"
        )
    payload = json.loads(json_path.read_text())
    resp = requests.post(
        url,
        json=payload,
        headers={"Authorization": f"Bearer {api_key}"},
        timeout=30,
    )
    resp.raise_for_status()
    log.info("uploaded blob: %s", resp.json())


def _parse_args(argv):
    # ... existing args ...
    parser.add_argument(
        "--upload",
        action="store_true",
        help="POST the local JSON to LEADS_UPLOAD_URL after writing it",
    )
    return parser.parse_args(argv)
```

## Locked-pending-redline decisions

1. **Stable blob key — `leads-current.json`, single key, overwrites.**
   Versioning deferred (out of scope per ROADMAP). React reads this one
   key. Trade-off: no rollback if a bad blob ships, but the upstream
   `data/leads.json` is committed to git, so worst case is "force-push
   blocked, manually re-upload yesterday's JSON." Acceptable for v1.
2. **Auth — bearer token in `Authorization` header**, not cookies, not
   query string. Token is `LEADS_UPLOAD_API_KEY` set in both Vercel
   project env and `pipeline/.env`. Generate via `openssl rand -hex 32`
   when configuring.
3. **Light body validation only** — top-level `niches` object with the
   3 required keys, each an array. Don't validate per-lead shape (the
   React UI is the consumer; if shape changes, both M9 and the validation
   need to evolve in lockstep — coupling is unhelpful).
4. **`access: 'public'`** on the Blob `put` — the blob is served via a
   public URL the React UI fetches client-side. The data is "outbound
   prospects we want to display"; nothing sensitive. Future
   authenticated read can flip to `'private'` + signed URLs.
5. **`addRandomSuffix: false` + `allowOverwrite: true`** — the SDK
   defaults to suffixing for deduplication. We *want* a stable URL the
   React UI can hardcode. Confirmed via `@vercel/blob` docs at planning
   time.
6. **Body size cap — 5 MB.** M7's smoke output is ~100 KB. 5 MB gives
   50× headroom before alerting. Vercel's default body parser is 1 MB,
   so we explicitly raise it via `export const config`.
7. **Method gate — POST only.** GET/PUT/etc → 405 with `Allow: POST`.
   No CORS preflight handling — pipeline calls server-to-server, no
   browser involvement.
8. **JS testing — none in M8.** Repo has no JS test infra (`react-scripts
   test` is for components, not API handlers). Verification is the
   manual smoke against a Vercel preview URL. If the function evolves
   in later milestones, revisit.
9. **Python `--upload` testing — mocked `requests.post`**. Verify URL,
   headers, body, and that exit code 1 surfaces on upload failure.
10. **Non-zero exit on upload failure** — M10 GitHub Actions cron uses
    process exit code to detect failure. The pipeline itself succeeded
    (DB and JSON were written); only the upload failed. Keep the local
    artifacts intact; let cron retry the upload on next run via
    `--upload` re-run, or the human can run `--upload` manually after
    fixing whatever broke.
11. **`requests` library** — already installed via M2 source modules
    (`requests>=2.31` in `pyproject.toml`). No new Python dep.
12. **Root `package.json` change — add `@vercel/blob`** as a runtime
    dependency. Also add a `.env.example` at the repo root documenting
    `BLOB_READ_WRITE_TOKEN` and `LEADS_UPLOAD_API_KEY` for the Vercel
    side. Do NOT touch `vercel.json` per root `CLAUDE.md`.
13. **`pipeline/.env.example` — add `LEADS_UPLOAD_URL` and
    `LEADS_UPLOAD_API_KEY`** for `daily_run.py --upload` to read. Drop
    `BLOB_READ_WRITE_TOKEN` from `pipeline/.env.example` (it was only
    "kept for parity"; M8 makes that fiction concrete — Blob writes
    happen in the Vercel function, not in Python).
14. **Commit granularity — single commit `add upload-leads api and
    --upload flag`.** Touches root + pipeline; bundle so the wiring is
    coherent.
