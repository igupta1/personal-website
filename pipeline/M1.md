# Plan — Milestone 1: `leads` schema + `db.py`

## Context

Per `pipeline/ROADMAP.md` M1: a SQLite `leads` table, Pydantic models, and a
small CRUD layer in `msp_pipeline/db.py` with fuzzy upsert. Schema first
because every downstream module reads or writes against it. Aim: simplest
shape that supports M2–M9 without ALTER TABLE later.

## Verified state

- `pipeline/msp_pipeline/` has only `__init__.py`. No utilities to reuse.
- `sqlite3` (stdlib), `pydantic 2.13`, `rapidfuzz 3.14` — installed in
  `pipeline/.venv`.

## Schema — `leads` table

```sql
CREATE TABLE IF NOT EXISTS leads (
    -- Identity
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT NOT NULL,
    name_key        TEXT NOT NULL UNIQUE,   -- normalized, for dedup
    domain          TEXT,                   -- when known

    -- Enrichment (nullable; populated by M4)
    industry        TEXT,
    headcount       INTEGER,
    country         TEXT,

    -- Signals (JSON array — see Pydantic below)
    signals         TEXT NOT NULL DEFAULT '[]',

    -- Per-niche scores (0-100 REAL; null until M5 scores)
    it_msp_score    REAL,
    mssp_score      REAL,
    cloud_score     REAL,

    -- Per-niche LLM copy (null until M6 generates)
    it_msp_insight    TEXT,
    mssp_insight      TEXT,
    cloud_insight     TEXT,
    it_msp_outreach   TEXT,
    mssp_outreach     TEXT,
    cloud_outreach    TEXT,

    -- Timestamps (UTC; updated_at bumped in app code)
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_leads_it_msp_score ON leads(it_msp_score DESC);
CREATE INDEX IF NOT EXISTS idx_leads_mssp_score   ON leads(mssp_score DESC);
CREATE INDEX IF NOT EXISTS idx_leads_cloud_score  ON leads(cloud_score DESC);
```

19 columns, 3 indexes (plus the implicit unique index on `name_key`).

**What I cut to keep this simple** (vs. the previous draft):
- Dropped `state`, `city`, `founded_year`, `linkedin_url`, `description`,
  `headcount_band`, `enriched_at`, `scored_at`, `outreach_generated_at`. Country
  is enough for v1 React filtering; `headcount_band` computed on read; one
  `updated_at` covers all pipeline-stage timestamps.
- Dropped `idx_leads_domain` and `idx_leads_industry`. SQLite full-scans 10k
  rows fast; add indexes when we measure a slow query.
- Single `updated_at` instead of splitting per stage.

## Models — `msp_pipeline/models.py`

```python
from datetime import datetime
from enum import Enum
from typing import Any
from pydantic import BaseModel, Field


class NicheName(str, Enum):
    IT_MSP = "it_msp"
    MSSP   = "mssp"
    CLOUD  = "cloud"


class SignalType(str, Enum):
    JOB_IT_SUPPORT              = "job_posted_it_support"
    JOB_IT_LEADERSHIP           = "job_posted_it_leadership"
    JOB_SECURITY                = "job_posted_security"
    JOB_CLOUD_DEVOPS            = "job_posted_cloud_devops"
    EXEC_HIRED                  = "exec_hired"
    FUNDING_RAISED              = "funding_raised"
    BREACH_DISCLOSED            = "breach_disclosed"
    HEADCOUNT_THRESHOLD_CROSSED = "headcount_threshold_crossed"
    HEADCOUNT_GROWTH_RAPID      = "headcount_growth_rapid"


class SourceName(str, Enum):
    JOBS = "jobs"; FUNDING = "funding"; BREACHES = "breaches"; COMPUTED = "computed"


class Signal(BaseModel):
    type: SignalType
    source: SourceName
    captured_at: datetime
    payload: dict[str, Any]   # source-specific; not validated centrally in v1


class Lead(BaseModel):
    """Mirrors a `leads` row. signals deserialized from JSON column."""
    id: int | None = None
    name: str
    name_key: str
    domain: str | None = None
    industry: str | None = None
    headcount: int | None = None
    country: str | None = None
    signals: list[Signal] = Field(default_factory=list)
    it_msp_score: float | None = None
    mssp_score: float | None = None
    cloud_score: float | None = None
    it_msp_insight: str | None = None
    mssp_insight: str | None = None
    cloud_insight: str | None = None
    it_msp_outreach: str | None = None
    mssp_outreach: str | None = None
    cloud_outreach: str | None = None
    created_at: datetime | None = None
    updated_at: datetime | None = None


class LeadCandidate(BaseModel):
    """What a source emits. db.upsert_lead turns this into a Lead row."""
    name: str
    domain: str | None = None
    initial_signal: Signal
```

## Open decisions for redline (4)

1. **Score range** — **0–100 REAL** (my pick). Allows fractional scoring, reads
   naturally in UI. Alternative: 0.0–1.0.
2. **Dedup strategy** — **fuzzy on `name_key`, threshold 90** (my pick). Handles
   breach data with no domain. Alternative: domain-first with name fallback.
3. **Signal payload shape** — **free-form `dict[str, Any]`** (my pick). Cheap;
   sources iterate without schema migrations. Alternative: typed Pydantic per
   `SignalType`. Defer until we have stable real-world payloads.
4. **Enrichment field set** — **`industry`, `headcount`, `country`** (my pick;
   only what the React UI and scoring need on day one). Add `state`/`city`/
   `description`/etc. later via `ALTER TABLE` if a milestone needs them.

  merges; "Acme Industries" inserts as a new row.

- `test_iter_leads_orders_by_niche_score` — three leads with different
  `it_msp_score`s; `iter_leads(niche=IT_MSP)` returns DESC, NULLs last.
- `test_update_lead_persists_and_bumps_updated_at` — write a few fields via
  allowlist; verify columns + `updated_at` advanced.
