# Plan — Milestone 3: `llm.py` wrappers (lean)

## Context

Per `pipeline/ROADMAP.md` M3: a single module that every LLM call in the
codebase routes through (per `pipeline/CLAUDE.md`: "All LLM calls go through
`msp_pipeline/llm.py` — no direct SDK use elsewhere"). Two providers
(Anthropic + Gemini), structured output via a Pydantic `response_model`,
exponential backoff on 429/5xx, and a logging hook so a malformed response is
debuggable.

Lean by design — no prompt templates here (those live in `enrichment.py` /
`outreach.py`), no dedicated `tests/test_llm.py` (M4 enrichment mocks the
wrapper and acts as its first real consumer).

## Module shape — `msp_pipeline/llm.py`

```python
from __future__ import annotations
import json, logging, os, time
from typing import TypeVar, overload
from dotenv import load_dotenv
from pydantic import BaseModel
from anthropic import Anthropic, APIStatusError, RateLimitError
from google import genai
from google.genai import types as genai_types

load_dotenv()  # idempotent; harmless if .env absent (CI sets vars directly)

log = logging.getLogger(__name__)

T = TypeVar("T", bound=BaseModel)

# Defaults — tunable per-call via kwarg; module constants set the floor.
_DEFAULT_CLAUDE_MODEL = "claude-haiku-4-5"   # cheapest current Claude; M6 may opt up to sonnet-4-6
_DEFAULT_GEMINI_MODEL = "gemini-2.5-flash"
_DEFAULT_MAX_TOKENS   = 1024
_MAX_RETRIES          = 4
_BACKOFF_BASE_S       = 1.5  # 1.5, 3, 6, 12s — capped at _MAX_RETRIES tries

class LLMError(RuntimeError):
    """Wrapper raised when both provider error semantics and our retries
    have been exhausted, or when structured output fails to validate."""

# --- Public surface ---------------------------------------------------------

@overload
def call_claude(prompt: str, *, response_model: type[T],
                model: str = ..., max_tokens: int = ...) -> T: ...
@overload
def call_claude(prompt: str, *, response_model: None = None,
                model: str = ..., max_tokens: int = ...) -> str: ...
def call_claude(prompt, *, response_model=None,
                model=_DEFAULT_CLAUDE_MODEL, max_tokens=_DEFAULT_MAX_TOKENS):
    """Anthropic wrapper. If response_model given, use forced tool-use to
    coerce to that Pydantic schema; else return the assistant's text. `model`
    overrides the cheap default (Haiku 4.5) — pass `claude-sonnet-4-6` from
    M6 for outreach copy if needed."""

@overload
def call_gemini(prompt: str, *, response_model: type[T]) -> T: ...
@overload
def call_gemini(prompt: str, *, response_model: None = None) -> str: ...
def call_gemini(prompt, *, response_model=None):
    """Gemini 2.5 Flash wrapper. Plain-text calls enable Google Search
    grounding via `tools=[Tool(google_search=GoogleSearch())]`. Structured
    calls (response_model set) skip grounding — the SDK rejects combining
    response_schema with the search tool."""
```

Both functions:
1. Build the request (provider-specific helper).
2. Run inside `_with_retries(...)` (see below).
3. Parse + validate; on validation failure, log the raw response and raise
   `LLMError`.

### Structured output — Anthropic via forced tool-use

Pydantic `model_json_schema()` → tool `input_schema`. One tool, name
`"return_<ModelName>"`. `tool_choice={"type": "tool", "name": ...}` forces
emission. Parse `response.content[0].input` (the tool_use block) through
`Model.model_validate(...)`.

### Structured output — Gemini via native schema

Pass `response_mime_type="application/json"` and `response_schema=Model`
inside `GenerateContentConfig`. `resp.parsed` is the Pydantic instance when
the SDK can validate; fallback path: `Model.model_validate_json(resp.text)`
with the raw text logged on validation failure.

### Text + grounding — Gemini

When `response_model is None`, build the config with
`tools=[Tool(google_search=GoogleSearch())]`. Grounding metadata
(`resp.candidates[0].grounding_metadata`) is logged at DEBUG but not
returned — callers get plain text. M4 enrichment doesn't need citations;
if M6 ever does, this returns to the API surface.

### Backoff + logging — `_with_retries`

```python
def _with_retries(fn, *, what: str):
    for attempt in range(_MAX_RETRIES):
        try:
            return fn()
        except (RateLimitError,) as e:
            wait = _BACKOFF_BASE_S * (2 ** attempt)
            log.warning("%s: rate limited (attempt %d/%d), sleep %.1fs: %s",
                        what, attempt + 1, _MAX_RETRIES, wait, e)
            time.sleep(wait)
        except APIStatusError as e:
            if 500 <= e.status_code < 600 and attempt < _MAX_RETRIES - 1:
                wait = _BACKOFF_BASE_S * (2 ** attempt)
                log.warning("%s: %d (attempt %d/%d), sleep %.1fs",
                            what, e.status_code, attempt + 1, _MAX_RETRIES, wait)
                time.sleep(wait)
                continue
            raise LLMError(f"{what}: non-retryable {e.status_code}") from e
    raise LLMError(f"{what}: exhausted {_MAX_RETRIES} retries")
```

Gemini errors are wrapped to the same shape via a small adapter
(`google.genai.errors.APIError` → branch on `.code`).

The "logging hook" the roadmap calls for is just `logging.getLogger(__name__)`
+ DEBUG-level log of the raw provider response on validation failure. Callers
opt in via standard `logging.basicConfig(level=logging.DEBUG)`.

## Locked decisions (post-redline)

1. **Provider swap: OpenAI gpt-4o-mini, no Claude.** Second user redline
   replaced Anthropic with OpenAI — Claude is not used in this project.
   Default model is `gpt-4o-mini` ($0.15/M in, $0.60/M out — ~6× cheaper
   than Haiku 4.5 on input). The wrapper exposes `call_openai` with the
   same surface that `call_claude` had: `(prompt, *, response_model=None,
   model="gpt-4o-mini", max_tokens=1024)`. M6 can opt up to `gpt-4o` or
   `gpt-4.1` per call if outreach quality needs it. Structured output via
   `client.beta.chat.completions.parse(response_format=PydanticModel)` —
   the SDK handles schema generation + validation.
2. **Default Gemini model — `gemini-2.5-flash` with Google Search grounding
   on the text path.** User confirmed grounding is desired. Wire grounding
   in via `tools=[Tool(google_search=GoogleSearch())]` for plain-text calls;
   skip it when `response_model` is set (the SDK rejects combining a JSON
   `response_schema` with the search tool — log a debug note and proceed
   without grounding in that path).
3. **Retry policy — handle in `llm.py`** (confirmed). Own exponential backoff
   (4 tries, 1.5/3/6/12s), our log shape. Don't lean on the SDKs' internal
   retries.
4. **Prompt caching — defer to M6** (confirmed). Cache only pays off with a
   stable per-niche system prompt; introduce `cache_control` in `outreach.py`
   where that prompt lives. Note: Haiku 4.5's minimum cacheable prefix is
   4096 tokens (per `shared/prompt-caching.md`), so M6 should size the
   per-niche system prompt accordingly when caching is added.
5. **Tests — none in M3** (confirmed). First exercise via mocked
   `call_claude` / `call_gemini` in `tests/test_enrichment.py` (M4). Manual
   smoke only in M3.
