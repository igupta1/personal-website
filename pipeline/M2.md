# Plan — Milestone 2: source modules

## Context

Per `pipeline/ROADMAP.md` M2: three source modules under
`msp_pipeline/sources/` that fetch from free APIs/feeds and return
`LeadCandidate` objects with attached `Signal`s. No DB writes from source
modules — they return candidates; `daily_run.py` (M7) does the upserts.
Tests use captured fixtures, never live HTTP (per `pipeline/CLAUDE.md`).

## Common module shape (across all 3)

```python
# msp_pipeline/sources/<source>.py

def fetch(*, since: datetime, limit: int | None = None) -> list[LeadCandidate]:
    """Aggregate per-API results. Per-API failures are logged and skipped;
    partial results returned. Optional limit caps the combined list."""

# Private, per-API (directly testable):
def _fetch_from_<api>(since: datetime) -> list[LeadCandidate]: ...
```

Mocking strategy in tests: patch the library boundary
(`jobspy.scrape_jobs`, `feedparser.parse`, `requests.get`) — never live HTTP.

## sources/jobs.py

Three APIs, all queried with the same module-level `_JOB_QUERIES` list
(IT support, security, cloud/DevOps, IT-leadership, exec keywords).

| API         | Library boundary                        | Time filter                        |
|-------------|-----------------------------------------|------------------------------------|
| JobSpy      | `jobspy.scrape_jobs(...)`               | `hours_old=(now - since)/h`        |
| Adzuna      | `requests.get(adzuna_search_url)`       | `max_days_old=(now - since)/d`     |
| HN Algolia  | `requests.get(hn_algolia_url)`          | `numericFilters=created_at_i>ts`   |

Adzuna requires `ADZUNA_APP_ID` + `ADZUNA_APP_KEY` from env; if missing,
`_fetch_from_adzuna` logs and returns `[]`.

For each posting, classify the title:

```python
def _classify_job_title(title: str) -> SignalType | None:
    # Order matters: exec wins ties.
    # Exec keywords (CIO/CISO/CTO/VP/Head of) → EXEC_HIRED
    # Security keywords          → JOB_SECURITY
    # Cloud/DevOps keywords      → JOB_CLOUD_DEVOPS
    # IT-leadership keywords     → JOB_IT_LEADERSHIP
    # IT-support keywords        → JOB_IT_SUPPORT
    # No match                   → None  (posting skipped)
```

One `Signal` per posting. Keyword lists are module-level constants
(`_EXEC_KEYWORDS`, `_SECURITY_KEYWORDS`, etc.); tunable later.

## sources/funding.py

Three sources:

| API              | Library boundary                              | Per-entry parse              |
|------------------|-----------------------------------------------|------------------------------|
| SEC EDGAR Form D | `requests.get(efts.sec.gov/.../search-index)` | filer-name → company         |
| TechCrunch RSS   | `feedparser.parse(techcrunch_feed_url)`       | feed-entry title → company   |
| PR Newswire RSS  | `feedparser.parse(prnewswire_feed_url)`       | feed-entry title → company   |

For RSS entries: **use the feed-entry title verbatim as the company name in
v1** — no regex extraction. M4 enrichment (with `llm.py`) can clean up later.
This keeps the source module focused on plumbing, not NLP.

All entries emit `SignalType.FUNDING_RAISED`. Entry timestamp from `published`
field; filter post-fetch by `>= since`.

## sources/breaches.py

Three sources, all HTML or CSV scrapes:

| Source            | Library boundary                          | Format       |
|-------------------|-------------------------------------------|--------------|
| HHS Wall of Shame | `requests.get(...)` + CSV parse           | CSV download |
| California AG     | `requests.get(...)` + BeautifulSoup       | HTML table   |
| Maine AG          | `requests.get(...)` + BeautifulSoup       | HTML table   |

Each entry: entity name → company, disclosure date → `captured_at`. All emit
`SignalType.BREACH_DISCLOSED`. Public-records data, no PII concerns.

## Fixtures

`pipeline/tests/fixtures/` layout:

```
tests/fixtures/

    sec_edgar.json
    techcrunch.xml
    prnewswire.xml
  breaches/
    hhs.csv

```

**Hand-crafted small samples** (3–5 entries each) modeled after the real API
shape. No live captures needed; faster to author and easier to maintain. If
the real API drift breaks parsers later, real captures can replace these
piecemeal.

A small helper in each test file: `_load_fixture(name) -> str | dict`.

  `test_classify_job_title` — table-driven cases covering all 5 signal types
  + a no-match.
