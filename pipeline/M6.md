# Plan — Milestone 6: `outreach.py`

## Context

Per `pipeline/ROADMAP.md` M6: for each niche where the lead's score crosses
a threshold, generate two pieces of LLM copy:

- **`insight`** — one sentence explaining why the lead is a fit for this
  niche (rendered in the React lead-card UI).
- **`outreach`** — a 3–5 sentence outreach email draft a salesperson can
  copy-paste and send.

Per-niche prompt templates, structured output via Pydantic. **Threshold
gating lives in M7** (`daily_run.py`); M6 is "given a lead, niche, score
— produce the copy." Pure function, no DB writes — consistent with M5's
`score()`.

## Verified state

- M1 schema has six writable copy columns:
  `it_msp_insight`, `mssp_insight`, `cloud_insight`,
  `it_msp_outreach`, `mssp_outreach`, `cloud_outreach`. All in
  `db._UPDATABLE_FIELDS`. No DB or schema change needed.
- M3 `llm.py` provides `call_openai(prompt, response_model=...,
  model=..., max_tokens=...)`. Default model `gpt-4o-mini`.
- M5 `scoring.score(lead)` produces `dict[NicheName, float]` in `[0, 100]`.
- M4 enrichment writes `industry`, `headcount`, `country`, plus
  `LOCATION_CAPTURED` signals carrying `{city, state}` payloads. M6 reads
  these for personalization.

## Module shape — `msp_pipeline/outreach.py`

```python
from __future__ import annotations
from datetime import datetime, timezone
from pydantic import BaseModel, Field
from msp_pipeline import llm
from msp_pipeline.models import Lead, NicheName, SignalType


class Copy(BaseModel):
    """Per-niche LLM copy pair. Public return type of `generate()`."""
    insight: str = Field(min_length=10, max_length=240)
    outreach: str = Field(min_length=80, max_length=1500)


# --- Per-niche framing ---------------------------------------------------

_NICHE_FRAMING: dict[NicheName, str] = {
    NicheName.IT_MSP: (
        "You write outbound copy for an IT managed service provider (MSP) "
        "that sells help desk, sysadmin, and general IT support to growing "
        "small and mid-sized businesses (10–250 employees)."
    ),
    NicheName.MSSP: (
        "You write outbound copy for a managed security service provider "
        "(MSSP) that sells SOC monitoring, vuln management, and compliance "
        "support to regulated SMBs (10–250 employees)."
    ),
    NicheName.CLOUD: (
        "You write outbound copy for a cloud consultancy that helps SMBs "
        "(10–250 employees) plan, migrate, and optimize their AWS / GCP / "
        "Azure footprint."
    ),
}


# --- Prompt template -----------------------------------------------------

_PROMPT_TEMPLATE = """\
{framing}

Lead profile:
- Company name: {name}
- Short form to use in the email body: "{short_name}"
- Industry: {industry}
- Approximate headcount: {headcount}
- Location: {location}
- Heat score for this niche: {score:.0f}/100

Recent signals (most recent first):
{signals}

Write two pieces of copy:
1. **insight** — ONE sentence (≤ 220 chars) explaining the most compelling
   reason this lead is a fit. Lead with the strongest recent signal,
   referencing concrete details from it (job title, funding round, etc.).
2. **outreach** — a 3–5 sentence outbound email body (no subject line, no
   greeting, no signature) that:
   - opens by naming a *specific* detail from one recent signal — the
     actual job title, the funding amount/round, the breach context,
     etc. (do NOT just say "I saw you posted a job" — say "I saw the
     Senior Security Engineer posting"),
   - addresses the company by the short form "{short_name}" (not the
     full legal name) — e.g. "Hope Q3 is going well at {short_name}",
   - frames how your service addresses what that signal implies,
   - ends with a soft, low-pressure CTA (e.g. a 15-min intro call).

Tone & style — write like a human salesperson, not like an AI:
- AVOID these AI-tell openers: "I hope this email finds you well",
  "I came across", "I noticed", "I wanted to reach out", "Just wanted
  to touch base", "I trust you're doing well".
- Prefer specific, conversational openers grounded in the signal.
- Professional B2B tone. No emojis. No exclamation marks. Contractions
  are fine ("you're", "we've").
- Do NOT fabricate facts not in the profile or signals — if a detail
  isn't there, don't invent it.

Special handling for breach signals:
- If a recent signal is a breach disclosure, treat it with EMPATHY,
  never urgency or alarm. The reader has already lived through the
  breach; the goal is to build trust as a future partner, not to
  exploit a vulnerable moment. Phrases like "we know how stressful a
  disclosure can be" or "no pitch — just wanted to introduce ourselves
  in case it's useful down the line" land better than urgency.
"""


# --- Public API ----------------------------------------------------------

def generate(
    lead: Lead,
    niche: NicheName,
    score: float,
    *,
    model: str = "gpt-4o-mini",
) -> Copy:
    """Generate (insight, outreach) for a single niche. One OpenAI call.
    Pure: does not write to the DB."""
    prompt = _PROMPT_TEMPLATE.format(
        framing=_NICHE_FRAMING[niche],
        name=lead.name,
        short_name=_short_name(lead.name),
        industry=lead.industry or "unknown",
        headcount=_describe_headcount(lead.headcount),
        location=_describe_location(lead),
        score=score,
        signals=_describe_signals(lead),
    )
    return llm.call_openai(prompt, response_model=Copy, model=model)


# --- Private helpers -----------------------------------------------------

_SCORING_SIGNAL_TYPES = frozenset({
    SignalType.JOB_IT_SUPPORT, SignalType.JOB_IT_LEADERSHIP,
    SignalType.JOB_SECURITY, SignalType.JOB_CLOUD_DEVOPS,
    SignalType.EXEC_HIRED, SignalType.FUNDING_RAISED,
    SignalType.BREACH_DISCLOSED,
})

_GENERIC_LEADING_WORDS = frozenset({"the", "a", "an"})

def _short_name(name: str) -> str:
    """Heuristic conversational form of a company name. e.g.
    'Angel's Marketing and Advertising Agency' -> 'Angel'.
    'The Acme Company' -> 'Acme'.
    'Stripe' -> 'Stripe'.
    The model is also told to use this form and may override if the
    heuristic produces something awkward."""
    tokens = name.strip().split()
    if not tokens:
        return name
    first = tokens[0]
    if first.lower() in _GENERIC_LEADING_WORDS and len(tokens) > 1:
        first = tokens[1]
    if first.endswith("'s") or first.endswith("’s"):
        first = first[:-2]
    return first

def _describe_headcount(n: int | None) -> str:
    if n is None:
        return "unknown"
    return f"~{n} employees"

def _describe_location(lead: Lead) -> str:
    """Pull the most recent LOCATION_CAPTURED payload, format as 'City, ST'.
    Falls back to country if location signal absent."""
    locs = sorted(
        (s for s in lead.signals if s.type == SignalType.LOCATION_CAPTURED),
        key=lambda s: s.captured_at, reverse=True,
    )
    if not locs:
        return lead.country or "unknown"
    payload = locs[0].payload
    city = payload.get("city")
    state = payload.get("state")
    if city and state: return f"{city}, {state}"
    if state:          return state
    if city:           return city
    return lead.country or "unknown"

# Per-signal-type payload field hints. Source modules (M2) put these into
# the payload dict; the prompt formatter reads what's present and skips
# what isn't. Keeps the formatter forgiving to payload schema drift.
_SIGNAL_PAYLOAD_FIELDS: dict[SignalType, tuple[str, ...]] = {
    SignalType.JOB_IT_SUPPORT:    ("title", "location"),
    SignalType.JOB_IT_LEADERSHIP: ("title", "location"),
    SignalType.JOB_SECURITY:      ("title", "location"),
    SignalType.JOB_CLOUD_DEVOPS:  ("title", "location"),
    SignalType.EXEC_HIRED:        ("title", "location"),
    SignalType.FUNDING_RAISED:    ("title", "amount_usd", "round"),
    SignalType.BREACH_DISCLOSED:  ("disclosed_on", "records_affected", "title"),
}

def _format_signal(sig: Signal, days_ago: int) -> str:
    """One bullet line for a signal — type + days-ago + payload details."""
    fields = _SIGNAL_PAYLOAD_FIELDS.get(sig.type, ())
    parts: list[str] = []
    for field in fields:
        val = sig.payload.get(field)
        if val:
            parts.append(f"{field}={val!r}")
    detail = f" — {'; '.join(parts)}" if parts else ""
    return f"- {sig.type.value} ({days_ago}d ago){detail}"

def _describe_signals(lead: Lead, *, limit: int = 6) -> str:
    """Most-recent N source signals as a bulleted list. Markers
    (LOCATION_CAPTURED, ENRICHMENT_RUN) excluded. Each line includes
    available payload details so the model can name specifics."""
    relevant = [s for s in lead.signals if s.type in _SCORING_SIGNAL_TYPES]
    relevant.sort(key=lambda s: s.captured_at, reverse=True)
    if not relevant:
        return "(no recent signals on file)"
    now = datetime.now(timezone.utc).replace(tzinfo=None)
    return "\n".join(
        _format_signal(s, max(0, (now - s.captured_at).days))
        for s in relevant[:limit]
    )
```

## Locked-pending-redline decisions

1. **Default model — `gpt-4o-mini`.** Same as enrichment classification.
   Cheap (~$0.0002/call), good enough for short B2B copy. M7 can override
   to `gpt-4o` per call if quality lacks. Alternative: default to `gpt-4o`
   from the start (~30× cost, marginal quality gain on 5-sentence emails).
2. **Single OpenAI call per niche per lead** (insight + outreach in one
   structured response). Alternative: two calls (insight then outreach,
   chained) — doubles cost for no benefit.
3. **Tone — human and conversational**, professional B2B, no emojis, no
   exclamation marks, contractions OK. The prompt also explicitly bans
   AI-tell openers ("I hope this finds you well", "I came across", "I
   noticed", "wanted to reach out", "touch base"). No fabricated facts.
   Hard-coded in the prompt — no per-niche tone overrides.
3a. **Address the company by short form** (e.g. "Angel" for "Angel's
   Marketing and Advertising Agency"). Heuristic computed in
   `_short_name(name)`: strip leading "the/a/an", drop apostrophe-s.
   Passed to the prompt as a separate variable; the model is told to
   prefer it in the email body.
3b. **Breach signals get empathy framing**, not urgency. Hard-coded in
   the prompt: "treat with EMPATHY... no pitch, just an introduction in
   case it's useful down the line." Avoid the appearance of exploiting a
   vulnerable moment — these are leads we want to convert into long-term
   clients, not single-incident sales.
4. **Signal injection — top 6 most-recent scoring signals with payload
   details.** Each bullet is `- <type> (Nd ago) — field=value; field=value`,
   e.g. `- job_posted_security (10d ago) — title='Senior Security
   Engineer'; location='Boston, MA'`. Per-signal-type field allowlist
   in `_SIGNAL_PAYLOAD_FIELDS` keeps formatter resilient to payload
   schema drift. Missing fields are silently skipped. Markers excluded.
5. **Location personalization — pull from most-recent
   `LOCATION_CAPTURED` payload**, fallback to `lead.country` (always "US"
   post-M4 redline). Alternative: skip location entirely (less
   personalized).
6. **Industry & headcount — included in prompt as context** even though
   they don't drive scoring. Both are useful for tone/framing ("a
   ~80-person legal firm"). Alternative: omit them (loses obvious
   personalization hooks).
7. **Score in prompt — included as `Heat score: NN/100`** so the model
   calibrates urgency. Alternative: omit (model infers from signal density).
8. **Public surface — `generate(lead, niche, score, *, model="gpt-4o-mini")
   -> Copy`** (Pydantic) instead of ROADMAP's `tuple[str, str]`. Pydantic
   gives field-name access in tests and downstream callers, and the
   structured-output round-trip already produces a Pydantic anyway.
9. **Pure function — no DB writes.** M7's `daily_run.py` calls
   `db.update_lead(it_msp_insight=..., it_msp_outreach=..., ...)`,
   gated on score-delta thresholds. Same pattern as `score()`.
10. **No re-generation skip logic in M6.** M7 is the orchestrator; it
    decides when to call `generate` based on score deltas (e.g.
    "regenerate if abs(new - old) > 10"). M6 always re-generates when
    called.
11. **Length constraints — Pydantic `Field(min_length=..., max_length=...)`**
    on both `Copy.insight` (10–240 chars) and `Copy.outreach` (80–1500
    chars). OpenAI structured output respects these constraints; if the
    model returns out-of-bounds copy the SDK raises before we see it.
12. **Prompt caching — defer.** OpenAI auto-caches prompts ≥1024 tokens
    when repeated; ours are ~600 tokens, below the threshold. The
    per-niche framing is a candidate for system-prompt caching when M10's
    nightly cron volume makes it worth wiring up. Note in out-of-scope.
13. **Commit granularity — single commit `add outreach.py`.** Pattern
    match.
14. **Iteration expected** — the user explicitly flagged that prompt
    quality will need tuning once real outputs are in front of them.
    Plan: ship M6 with the constraints above, do the manual smoke at
    the bottom of "Verification" against 3–5 real leads per niche,
    iterate on the prompt + framings + length bounds based on what reads
    well, then re-run the smoke. Don't try to nail the prompt in one
    shot — get it in front of the user instead.

## Tests — `tests/test_outreach.py`

11 tests:

- **`test_describe_headcount`** — `None → "unknown"`, `120 → "~120 employees"`.
- **`test_short_name`** — table-driven:
  `"Angel's Marketing and Advertising Agency" → "Angel"`,
  `"The Acme Company" → "Acme"`, `"Stripe" → "Stripe"`,
  `"Pioneer Legal LLP" → "Pioneer"`, `"" → ""`,
  `"Angel’s Studio" → "Angel"` (curly apostrophe).
- **`test_describe_location_uses_latest_signal`** — lead with two
  `LOCATION_CAPTURED` signals at different `captured_at`; expect the
  newer payload's `city, state` returned.
- **`test_describe_location_falls_back_to_country`** — no
  `LOCATION_CAPTURED` signals → returns `lead.country`. Truly empty
  (no country) → `"unknown"`.
- **`test_describe_signals_excludes_markers`** — lead with one source
  signal + one `LOCATION_CAPTURED` + one `ENRICHMENT_RUN`; output
  contains only the source signal line.
- **`test_describe_signals_includes_payload_fields`** — lead with a
  `JOB_SECURITY` signal carrying
  `payload={"title": "Senior Security Engineer", "location": "Boston, MA"}`;
  output line contains both `title=` and `location=` substrings (so the
  model can name them in the email).
- **`test_describe_signals_skips_missing_payload_fields`** — same signal
  type but `payload={}`; output line is just `<type> (Nd ago)` with no
  trailing detail clause.
- **`test_describe_signals_limits_and_orders`** — 10 signals across types
  and dates; expect top 6 most-recent in newest-first order.
- **`test_generate_calls_openai_with_full_prompt`** — mock
  `outreach.llm.call_openai` (capture call args), verify the prompt
  contains: lead name, the short-form name, industry, headcount snippet,
  niche-specific framing keyword (e.g. "managed security service provider"
  for MSSP), score string, the AI-tell ban phrase ("I hope this email
  finds you well"), the breach-empathy phrase ("EMPATHY"), and at least
  one signal line; verify `response_model is Copy` and
  `model == "gpt-4o-mini"`.
- **`test_generate_returns_copy_pydantic`** — mock `call_openai` to
  return `Copy(insight="x"*30, outreach="y"*200)`; verify
  `generate()` returns the same `Copy` instance.
- **`test_generate_per_niche_framing_differs`** — call `generate` once
  per niche with the same lead, capture each prompt; assert the three
  prompts are distinct and each contains the niche-specific framing
  string from `_NICHE_FRAMING`.

Mocking strategy: `monkeypatch.setattr(outreach.llm, "call_openai", ...)`
— same pattern as `test_enrichment.py`.

## Out of scope (per `pipeline/ROADMAP.md`)

- Multi-language copy / localization.
- A/B testing different prompts or models — defer until M10 cron volume
  produces enough sent emails to compare reply rates.
- Embeddings / RAG over a CRM — outreach in v1 is purely
  signal-templated; no retrieval.
- Subject lines and greeting/signature scaffolding — deliberately omitted
  from the prompt so reps can paste the body into their own templates.
- Score-delta gating for re-generation — lives in M7 `daily_run.py`.
- Prompt caching — OpenAI's auto-cache needs ≥1024-token prompts; ours
  are ~600. Wire this up only if M10 cron volume justifies cache rewrite
  cost.
- Persistence — `generate()` is pure; M7 writes the columns.
