# Plan — Milestone 7: `daily_run.py` + JSON output (local)

## Context

Per `pipeline/ROADMAP.md` M7: orchestrate the full pipeline end-to-end
against a real DB. Fetch → upsert → enrich new/stale → re-score →
regenerate copy where the score moved past a delta → emit
`pipeline/data/leads.json` for the React UI to consume.

This is **plumbing**, not new logic. Every piece of business logic already
lives in M2–M6. M7's job is to wire them together, isolate per-source
failures, gate LLM calls on cost-saving thresholds, and emit a JSON shape
M8/M9 can consume.

Vercel upload is M8; React pages are M9.

## Verified state

- M2 source modules:
  `jobs.fetch(*, since: datetime, limit: int | None = None) -> list[LeadCandidate]`,
  same shape for `funding.fetch` and `breaches.fetch`.
- M1 `db`:
  `init_db(path) -> Connection`, `upsert_lead(conn, candidate)`,
  `iter_leads(conn, *, niche=None, min_score=None, limit=None)`,
  `update_lead(conn, lead_id, **fields)`, `get_lead(conn, ...)`,
  `delete_lead(conn, lead_id)`, `append_signal(conn, lead_id, sig)`.
- M4 `enrichment.enrich(conn, lead, *, force=False) -> bool` (returns
  False if lead was deleted as non-US/oversized).
- M5 `scoring.score(lead, *, now=None) -> dict[NicheName, float]` (pure).
- M6 `outreach.generate(lead, niche, score, *, model='gpt-4o-mini') -> Copy`
  (pure).
- `pipeline/data/leads.db` is committed; `data/leads.json` is the new
  artifact this milestone produces (also committed so the cron in M10
  can update it incrementally).

## Module shape — `msp_pipeline/daily_run.py`

```python
from __future__ import annotations
import argparse, json, logging, sqlite3, sys
from datetime import datetime, timedelta, timezone
from pathlib import Path
from typing import Any
from msp_pipeline import db, enrichment, outreach, scoring
from msp_pipeline.models import Lead, LeadCandidate, NicheName, SignalType
from msp_pipeline.sources import breaches, funding, jobs

log = logging.getLogger("daily_run")

DEFAULT_DB_PATH      = Path("data/leads.db")
DEFAULT_OUTPUT_PATH  = Path("data/leads.json")
LOOKBACK_DAYS        = 14   # window passed to source.fetch(since=...)
COPY_THRESHOLD       = 40.0 # min score to (re)generate copy for a niche
COPY_DELTA_THRESHOLD = 10.0 # regenerate if abs(new - old) > this
# JSON output is unfiltered per redline — every lead in the DB ships, sorted
# DESC by per-niche score with NULL scores last (db.iter_leads handles this).
# UI is the filtering surface. Add a cap here if Vercel Blob payload size
# becomes a concern.


def main(argv: list[str] | None = None) -> int:
    args = _parse_args(argv)
    logging.basicConfig(
        level=logging.DEBUG if args.verbose else logging.INFO,
        format="%(asctime)s %(name)s %(levelname)s %(message)s",
    )
    if args.dry_run:
        log.info("dry-run: skipping all writes (DB, LLM calls, JSON)")

    candidates = _fetch_all(since=_utcnow() - timedelta(days=LOOKBACK_DAYS),
                            per_source_limit=args.limit)
    log.info("fetched %d candidates across all sources", len(candidates))
    if args.dry_run:
        return 0

    conn = db.init_db(args.db_path)
    upserted_ids = _upsert_all(conn, candidates)
    log.info("upserted %d leads", len(upserted_ids))

    enriched_ids = _enrich_all(conn, upserted_ids, force=args.reenrich)
    log.info("kept %d enriched leads (others deleted as non-US/oversized)",
             len(enriched_ids))

    rescored_ids, copy_calls = _rescore_and_regen_copy(
        conn, enriched_ids, model=args.copy_model
    )
    log.info("re-scored %d leads, generated copy %d times",
             len(rescored_ids), copy_calls)

    output = _build_output(conn)
    args.output_path.write_text(json.dumps(output, indent=2, default=str))
    log.info("wrote %s (%d niche slices)", args.output_path,
             len(output["niches"]))
    return 0


# --- Stages -----------------------------------------------------------

def _fetch_all(*, since: datetime, per_source_limit: int | None
               ) -> list[LeadCandidate]:
    """Run all 3 fetchers sequentially. Per-source failures are logged
    and swallowed; surviving candidates returned."""
    out: list[LeadCandidate] = []
    for name, fn in [("jobs", jobs.fetch),
                     ("funding", funding.fetch),
                     ("breaches", breaches.fetch)]:
        try:
            cs = fn(since=since, limit=per_source_limit)
            log.info("source %s returned %d candidates", name, len(cs))
            out.extend(cs)
        except Exception as e:
            log.exception("source %s failed: %s", name, e)
    return out

def _upsert_all(conn, candidates) -> list[int]:
    ids: list[int] = []
    for c in candidates:
        try:
            lead = db.upsert_lead(conn, c)
            assert lead.id is not None
            ids.append(lead.id)
        except Exception:
            log.exception("upsert failed for candidate %r", c.name)
    # Dedup; one lead can be hit multiple times in a single run.
    return list(dict.fromkeys(ids))

def _enrich_all(conn, lead_ids, *, force: bool) -> list[int]:
    kept: list[int] = []
    for lead_id in lead_ids:
        lead = db.get_lead(conn, lead_id=lead_id)
        if lead is None:
            continue  # deleted by an earlier enrich call this run
        try:
            if enrichment.enrich(conn, lead, force=force):
                kept.append(lead_id)
        except Exception:
            log.exception("enrichment failed for lead %s", lead.name)
            kept.append(lead_id)  # keep the row; try again tomorrow
    return kept

def _rescore_and_regen_copy(conn, lead_ids, *, model: str) -> tuple[list[int], int]:
    """Re-score every lead (cheap). For each niche where the new score
    crosses COPY_THRESHOLD AND moved more than COPY_DELTA_THRESHOLD
    against the persisted score, regenerate copy via outreach.generate."""
    rescored: list[int] = []
    copy_calls = 0
    for lead_id in lead_ids:
        lead = db.get_lead(conn, lead_id=lead_id)
        if lead is None:
            continue
        new_scores = scoring.score(lead)
        score_updates: dict[str, Any] = {}
        copy_updates: dict[str, Any] = {}
        for niche, new in new_scores.items():
            score_col = _NICHE_SCORE_COL[niche]
            old = getattr(lead, score_col)
            score_updates[score_col] = new
            should_regen = (
                new >= COPY_THRESHOLD
                and (old is None or abs(new - (old or 0.0)) > COPY_DELTA_THRESHOLD)
            )
            if should_regen:
                try:
                    copy = outreach.generate(lead, niche, new, model=model)
                    copy_updates[_NICHE_INSIGHT_COL[niche]] = copy.insight
                    copy_updates[_NICHE_OUTREACH_COL[niche]] = copy.outreach
                    copy_calls += 1
                except Exception:
                    log.exception("outreach.generate failed for %s / %s",
                                  lead.name, niche.value)
        db.update_lead(conn, lead_id, **score_updates, **copy_updates)
        rescored.append(lead_id)
    return rescored, copy_calls


# --- JSON output ------------------------------------------------------

def _build_output(conn) -> dict[str, Any]:
    """JSON shape consumed by api/generate-leads.js (M8) and the React
    pages (M9). One slice per niche; every lead in the DB appears in
    every slice, sorted by that niche's score DESC with NULL scores last
    (db.iter_leads enforces this). Filtering is the UI's job."""
    return {
        "generated_at": _utcnow().isoformat(),
        "niches": {
            niche.value: [
                _lead_to_json(lead, niche)
                for lead in db.iter_leads(conn, niche=niche)
            ]
            for niche in NicheName
        },
    }

def _lead_to_json(lead: Lead, niche: NicheName) -> dict[str, Any]:
    """Per-niche projection. Score / insight / outreach are pulled from
    the niche-specific columns. City/state pulled from the most-recent
    LOCATION_CAPTURED signal payload. Signal list shows only the top
    6 most-recent scoring signals (no markers)."""
    ...  # builds the per-card dict the React UI renders
```

## CLI flags

```
python -m msp_pipeline.daily_run [options]

  --db-path PATH        SQLite path (default: data/leads.db)
  --output-path PATH    JSON path (default: data/leads.json)
  --limit N             Per-source candidate cap (default: no limit)
  --reenrich            Force re-enrichment on every lead (default: skip if
                        already enriched and no new signals; see M4 #5)
  --copy-model MODEL    OpenAI model for outreach copy (default: gpt-4o-mini)
  --dry-run             Run sources only; skip DB writes, LLM calls, JSON
                        write. Useful for verifying source connectivity.
  --verbose             DEBUG-level logging.
```

## Locked-pending-redline decisions

1. **Sequential source fetch** (jobs → funding → breaches). One process,
   one DB connection. Concurrent fetching adds complexity for marginal
   gain at our volume. Per-source failure isolation already covered by
   try/except around each `fn(since=...)`.
2. **Lookback window — 14 days.** Source modules use `since=now - 14d`.
   Long enough to catch postings/funding/breaches we missed in any
   recent skipped runs; short enough that the per-run candidate set
   stays manageable. Alternative: persist `last_run_at` per source
   (premature; the upsert dedup makes overlap cheap).
3. **Re-score every lead every run.** Scoring is pure-Python and cheap;
   recency decay shifts scores even without new signals. Alternative:
   only re-score touched leads (loses decay-driven score drift).
4. **Copy regeneration gating — `score >= 40` AND
   `abs(new - old) > 10`.** Both conditions required. Prevents
   regenerating copy for cold leads (saves ~$0.0006/lead/niche) and
   prevents regenerating on tiny score wiggles. First-time crossing
   `40` (when `old is None`) always generates. Numbers are guesses,
   tune in M10.
5. **JSON shape — one file, three niche slices, every lead included.**
   Per redline: ship everything; UI sorts (least-qualified at the
   bottom, NULLs last) and can add visibility cutoffs later when the
   data is in front of us. `db.iter_leads(conn, niche=niche)` already
   sorts DESC by `<niche>_score` with NULLs last, so the JSON is ready
   to render verbatim. Add a cap if Vercel Blob payload size becomes a
   problem — none expected at SMB-targeting volumes.
6. **Per-source / per-lead failure handling — log-and-continue.** No
   retries within a run. The next nightly run re-fetches and the upsert
   merges. Alternative: in-run retries (premature; the job is daily,
   not real-time).
7. **`--dry-run` = no writes anywhere.** Sources are called, candidates
   are counted and logged, then the function returns. Useful for
   verifying `OPENAI_API_KEY`/`GEMINI_API_KEY`/`ADZUNA_*` plumbing
   without spending money.
8. **`--limit` = per-source cap**, not total. Easier to reason about
   when one source is wonky.
9. **Output JSON includes only top-6 most-recent scoring signals
   per lead** (not the full signal history). Matches what `outreach.py`
   shows the LLM. Markers (`LOCATION_CAPTURED`, `ENRICHMENT_RUN`)
   excluded. Keeps the JSON small.
10. **CLI logging — stdlib `logging`, INFO by default, DEBUG with
    `--verbose`.** GitHub Actions cron (M10) will capture stdout/stderr;
    no need for a structured-log library here.
11. **Tests — full pipeline with all sources mocked + the LLM wrappers
    mocked.** Real DB (tmp_path SQLite). No live HTTP, no real LLM
    calls. Aim: assert the end-to-end shape (candidates → upsert →
    enrich → score → copy → JSON write) and the gating logic.
12. **Commit granularity — single commit `add daily_run.py`.**

## Tests — `tests/test_daily_run.py`

7 tests:

- **`test_main_dry_run_makes_no_writes`** — mock all 3 source `fetch`
  functions to return small candidate lists; mock `enrichment.enrich`
  and `outreach.generate` to assert they're never called; run
  `main(["--dry-run"])`; assert exit 0, no DB file created, no JSON
  file created.
- **`test_main_end_to_end_smoke`** — mock the 3 sources to return 2
  candidates each (6 total). Mock `enrichment.lookup_company` (one
  US/120-employee, one non-US) and `enrichment.classify_industry`. Mock
  `outreach.generate` to return a stub `Copy`. Run `main([])` against
  a `tmp_path` DB and JSON path. Assert: JSON exists, has all 3 niche
  slices, at least one slice non-empty, every lead in the JSON has the
  required keys (per `_lead_to_json` schema).
- **`test_per_source_failure_isolated`** — mock one source to raise
  `RuntimeError`, the other two to return candidates. Run main; assert
  the surviving candidates were upserted and the failure was logged.
- **`test_copy_regen_threshold_gates_calls`** — pre-seed two leads in
  the DB: lead A has `it_msp_score=20` (below threshold), lead B has
  `it_msp_score=50` with no insight/outreach yet. After mocked
  re-scoring puts both above 40, assert `outreach.generate` was called
  once for B (first crossing) and **not** for A. Then re-run with
  scores nudged by less than the delta; assert no further `generate`
  calls.
- **`test_json_output_shape`** — pre-seed leads with known scores
  (including one with `it_msp_score=None`); invoke `_build_output(conn)`
  directly; assert: top-level `generated_at` and `niches` keys, three
  niche keys, every lead appears in every niche slice, leads sorted
  DESC by that niche's score with NULL-scored leads at the bottom (no
  cutoff, no cap).
- **`test_lead_to_json_signals_filtered_and_capped`** — lead with 10
  scoring signals + 1 marker; assert the JSON's `signals` list has
  length 6 (cap) and excludes the marker.
- **`test_lead_to_json_pulls_city_state_from_signal`** — lead with a
  `LOCATION_CAPTURED` signal in `payload={"city": "Boston",
  "state": "MA"}`; assert JSON has `city` and `state` populated. Lead
  without that signal → both fields `null`.

Mocking strategy: `monkeypatch.setattr` on the names imported into
`daily_run`'s namespace (e.g. `daily_run.jobs.fetch`,
`daily_run.enrichment.enrich`, `daily_run.outreach.generate`).
