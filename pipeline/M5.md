# Plan — Milestone 5: `scoring.py` (signal-only)

## Context

Per `pipeline/ROADMAP.md` M5: deterministic per-niche scorer in `[0, 100]`.
**No LLM** — pure rule-based, explainable, reproducible.

**Per the M5 redline session, scoring is signal-only.** Industry and
headcount are *not* score inputs — they're UI filters. The user filtering
the React pages can already say "show me only legal-industry leads with
51–200 employees"; the score answers the separate question "given everything
that's happened to this company recently, how hot is the lead?"

A second redline pulled the SMB cap (`headcount > 250`) up into M4
`enrich()` (locked decision #8 there). M5 itself doesn't see leads with
>250 employees because they're already deleted.

## Verified state

- M4 wrote `industry` and `headcount` (≤ 250) and `country` ("US"). M5
  reads `lead.signals` only.
- `lead.signals` carries source signals (`JOB_*`, `EXEC_HIRED`,
  `FUNDING_RAISED`, `BREACH_DISCLOSED`) plus enrichment markers
  (`LOCATION_CAPTURED`, `ENRICHMENT_RUN`). Markers aren't keyed in any
  niche's weight table, so they naturally contribute zero.
- `db._UPDATABLE_FIELDS` already includes `it_msp_score`, `mssp_score`,
  `cloud_score`. Nothing in `db.py` needs to change.

## Module shape — `msp_pipeline/scoring.py`

```python
from __future__ import annotations
from datetime import datetime, timezone
from msp_pipeline.models import Lead, NicheName, SignalType

# Recency decay: exponential, 30-day half-life. Yesterday ~100%, 30 days
# ago 50%, 90 days ago 12.5%.
HALF_LIFE_DAYS = 30.0

# Per-niche signal weights. Only listed signal types contribute. Each
# signal's contribution is `weight × decay`. Sum is clamped to [0, 100].
# Magnitudes are tuned so a single fresh top signal lands a lead in the
# "promising" range (~35–45), 2–3 fresh top signals lands "hot" (80+),
# and full-spectrum activity caps at 100.
SIGNAL_WEIGHTS: dict[NicheName, dict[SignalType, float]] = {
    NicheName.IT_MSP: {
        SignalType.JOB_IT_LEADERSHIP: 35,
        SignalType.JOB_IT_SUPPORT:    25,
        SignalType.FUNDING_RAISED:    25,
        SignalType.EXEC_HIRED:        18,
        SignalType.BREACH_DISCLOSED:  18,
        SignalType.JOB_SECURITY:      12,
        SignalType.JOB_CLOUD_DEVOPS:  12,
    },
    NicheName.MSSP: {
        SignalType.BREACH_DISCLOSED:  45,
        SignalType.JOB_SECURITY:      40,
        SignalType.FUNDING_RAISED:    22,
        SignalType.JOB_IT_LEADERSHIP: 18,
        SignalType.EXEC_HIRED:        15,
        SignalType.JOB_CLOUD_DEVOPS:  12,
        SignalType.JOB_IT_SUPPORT:    10,
    },
    NicheName.CLOUD: {
        SignalType.JOB_CLOUD_DEVOPS:  40,
        SignalType.FUNDING_RAISED:    28,
        SignalType.JOB_IT_LEADERSHIP: 18,
        SignalType.EXEC_HIRED:        15,
        SignalType.JOB_SECURITY:      12,
        SignalType.JOB_IT_SUPPORT:    10,
        SignalType.BREACH_DISCLOSED:   8,
    },
}

SCORE_MAX = 100.0
SCORE_MIN = 0.0


def score(lead: Lead, *, now: datetime | None = None) -> dict[NicheName, float]:
    """Per-niche score in [0, 100]. ``now`` is injectable for deterministic
    tests; defaults to UTC current time. Pure function — does not write."""
    now = now or datetime.now(timezone.utc).replace(tzinfo=None)
    return {niche: _score_niche(lead, niche, now) for niche in NicheName}


# Private --------------------------------------------------------------

def _score_niche(lead: Lead, niche: NicheName, now: datetime) -> float:
    weights = SIGNAL_WEIGHTS[niche]
    total = 0.0
    for sig in lead.signals:
        w = weights.get(sig.type)
        if w is None:
            continue
        total += w * _decay(sig.captured_at, now)
    return max(SCORE_MIN, min(SCORE_MAX, total))

def _decay(captured_at: datetime, now: datetime) -> float:
    days = max(0.0, (now - captured_at).total_seconds() / 86400.0)
    return 0.5 ** (days / HALF_LIFE_DAYS)
```

## Locked decisions

1. **Score = signal sum, clamped to [0, 100]** (post-redline). No
   industry-fit or headcount-fit components. Industry and headcount are
   stored on the lead but used as UI filters in M9, not as score inputs.
2. **Recency decay — exponential, 30-day half-life** (confirmed). Yesterday
   ~100%, 30 days 50%, 90 days 12.5%.
3. **Missing data — score what we can** (confirmed). A lead with no
   contributing signals scores 0 across all niches. No penalty math.
4. **Cap = 100, applied as a clamp** (post-redline simplification — no
   separate `SIGNAL_CAP=50` since signals are now the entire score). A
   spammy lead with 30 fresh job postings can saturate at 100 — that's
   correct behavior since the signals are real, not noise.
5. **`SignalType.LOCATION_CAPTURED` and `ENRICHMENT_RUN` ignored** — not
   in any niche's `SIGNAL_WEIGHTS` table; `weights.get(sig.type)` returns
   `None` and the signal is skipped.
6. **`score(lead)` is pure — does NOT write to the DB.** The orchestrator
   (`daily_run.py` in M7) calls `db.update_lead(it_msp_score=..., ...)`
   itself, gated on a delta threshold.
7. **`now` is injectable** for deterministic tests. Defaults to
   `datetime.now(utc).replace(tzinfo=None)` matching the codebase's
   `_utcnow()` convention.
8. **Weight numbers are initial guesses, expected to be tuned.** Calibrated
   so single fresh top-signal ~35–45, 2–3 fresh top-signals ~80+, full
   spectrum caps at 100. Will need real-world tuning once M10 cron
   produces enough scored leads.
9. **Commit granularity — single commit `add scoring.py`** (plus the
   small enrichment.py SMB-cap addition that landed in the same M5 redline
   round). Same pattern as M2/M3/M4.

## Tests — `tests/test_scoring.py`

8 tests:

- **`test_score_returns_three_niches`** — empty-signal lead → dict with
  the three `NicheName` keys, all 0.
- **`test_score_zero_for_blank_lead`** — `Lead(name="x", name_key="x")`
  with no signals → all three scores == 0.
- **`test_single_fresh_signal_scores_full_weight`** — lead with one
  `BREACH_DISCLOSED` signal `captured_at == now` → mssp score == 45
  (weight, no decay), it_msp == 18, cloud == 8. Within float tolerance.
- **`test_recency_decay_halves_at_one_half_life`** — same lead+signal,
  but `captured_at = now - 30 days`; assert score is half of fresh-case
  score (within 1% tolerance).
- **`test_recency_decay_floors_at_zero_for_future_dates`** — pathological
  case: signal with `captured_at > now` (clock skew, test fixture) →
  decay clamps at 1.0 (no negative-days exponent that would *amplify*
  the weight).
- **`test_score_clamps_to_100`** — lead with all 7 signal types, all
  fresh → all three niche scores == 100 exactly (raw sum exceeds the
  cap; clamp must trigger). Verify `<= 100` strictly.
- **`test_unknown_signal_types_ignored`** — lead with only
  `LOCATION_CAPTURED` and `ENRICHMENT_RUN` signals → scores all 0.
- **`test_score_is_pure_no_db_writes`** — run `score(lead)` against a
  lead loaded from a tmp_path SQLite; reload the lead from DB; assert
  all score columns are still NULL (we did not accidentally persist).

All tests pass `now=` explicitly so decay math is deterministic.
